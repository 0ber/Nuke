Nuke

Advanced framework for loading, processing, caching and preheating images

{% highlight swift %} var request = ImageRequest(URL: NSURL(string: "http://...")) request.targetSize = CGSize(width: 200, height: 200) // Resize image request.processor = ImageFilterGaussianBlur() // Apply image filters Nuke.taskWith(request) { response in let image = response.image }.resume() {% endhighlight %}
Easy to Use

Zero config and user-friendly. The goal is simplicity. Without compromising on extensibility.

Fast

Performant, asynchronous, and fully thread-safe.

Extensible

Nuke is a pipeline with injectable dependencies. Has Alamofire and FLAnimatedImage plugins.

Features

Loading

  • Uses NSURLSession with HTTP/2 support
  • Install Alamofire plugin or integrate your own networking with a couple lines of code
  • A single data task is used for multiple equivalent requests
  • Congestion control that prevents trashing the system with requests during fast scrolling
  • Automated preheating of images close to the viewport
  • Full featured extensions for UI components

Caching

  • Doesn't reinvent caching, relies on HTTP cache in URL Loading System
  • Two cache layers including auto-purging memory cache for processed images
  • Intellegent control over memory cache

Processing

  • Create, compose and apply image filters
  • Core Image integration
  • Background image decompression and scaling in a single step
  • Resize loaded images to fit displayed size

Installation

Supported platforms: iOS 8.0+ / watchOS 2.0+ / OS X 10.9+ / tvOS 9.0+

CocoaPods

To install Nuke add a dependency to your Podfile:

{% highlight ruby %} # source 'https://github.com/CocoaPods/Specs.git' # use_frameworks! # platform :ios, "8.0" / :watchos, "2.0" / :osx, "10.9" / :tvos, "9.0" pod "Nuke" pod "Nuke-Alamofire-Plugin" # optional pod "Nuke-AnimatedImage-Plugin" # optional {% endhighlight %}
